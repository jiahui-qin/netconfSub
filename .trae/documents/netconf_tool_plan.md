# Netconf本地工具实现计划（重构版）

## 1. 项目现状分析

### 1.1 现有功能
- 基于Spring Boot的Java项目
- 已实现基本的Netconf连接功能（Device类）
- 已实现Netconf会话管理（NetconfSession类）
- 已实现基本的Netconf操作（配置获取、修改、提交等）
- 已实现RPC执行功能

### 1.2 重构需求
- 放弃Java实现，采用现代化前端技术栈
- 设计现代化、用户友好的界面
- 支持在一个页面中查看订阅消息、Netconf XML发送和接收
- 提供直观的操作体验

## 2. 技术栈选择

### 2.1 前端技术
- **框架**：React + Vite
- **UI库**：Tailwind CSS + shadcn/ui
- **状态管理**：React Context + useReducer
- **网络请求**：Axios
- **WebSocket**：用于实时通知

### 2.2 后端技术
- **语言**：Node.js
- **框架**：Express.js
- **Netconf库**：node-netconf-client
- **WebSocket**：socket.io

### 2.3 构建工具
- **前端**：Vite
- **后端**：Node.js + npm

## 3. 架构设计

### 3.1 系统架构
```
┌─────────────────────┐    WebSocket    ┌─────────────────────┐
│                     │◄────────────────┤                     │
│  前端应用 (React)   │                │  后端服务 (Node.js) │
│                     │────────────────►│                     │
└─────────────────────┘    HTTP API    └─────────────────────┘
                                          │
                                          │ Netconf
                                          ▼
                                  ┌─────────────────────┐
                                  │  Netconf设备        │
                                  └─────────────────────┘
```

### 3.2 核心模块

#### 3.2.1 前端模块
- **连接管理**：设备连接的创建、管理和状态监控
- **消息管理**：XML消息的发送、接收和展示
- **通知订阅**：通知的订阅、接收和展示
- **配置管理**：设备配置的获取和修改

#### 3.2.2 后端模块
- **Netconf客户端**：处理与设备的Netconf通信
- **WebSocket服务**：实时推送通知和消息
- **API服务**：提供HTTP接口供前端调用
- **连接池管理**：管理多个Netconf连接

## 4. 界面设计

- **现代化**：采用现代UI设计，响应式布局
